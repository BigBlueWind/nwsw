EnodeB Registration

Base Station Identifier: Base Station Identifier. Used for identifying the BS before EMS registration.

Public DNS IP Address: IPv4 address of the primary public DNS Server

Public DNS IP Address: IPv4 address of the secondary public DNS Server

Private DNS IP Address: IPv4 address of the primary private DNS Server

Private DNS IP Address: IPv4 address of the secondary private DNS Server

Carrier Security GW Address: IPv4 address of the primary security GW the Pico eNB attempts to establish connection with.

Carrier Security GW Address: IPv4 address of the secondary security GW the Pico eNB attempts to establish connection with.

EMS IP Address: IPv4 address of EMS.

Latitude: Latitude configured into ENB. To be used if timeAndLocationSourceToUse is set to 'operConfigured' or when not available from GPS. Format is MMNN.PPPPPP(N/S). MM -> Degrees, NN -> Minutes, PPPPPP -> Decimal precision, North (N) or South (S).

Longitude: Longitude configured into ENB. To be used if timeAndLocationSourceToUse is set to 'operConfigured' or when not available from GPS. Format is MMMNN.PPPPPP(E/W). MMM -> Degrees, NN -> Minutes, PPPPPP -> Decimal precision, East (E) or West (W).

SNMP Community String: The community string the ENB should accept in SNMP PDUs. Other packets will be dropped. ENB will use the same in outgoing PDUs.

Time And Location Source To Use: The time and location source configured to be used by ENB: gps : Use GPS based time (and location) for setting ENB clock operConfigured : Use operator configured time (and location) for setting ENB clock

Initial Local Time: It is a MANDATORY field, since this can be used as a fallback while GPS fails to lock/respond. If configured by Operator and ENB is configured to use this for date and time setting, ENB will use this for setting date and time of the ENB at initialization. Otherwise ENB will try to use other source GPS. YYYYMMDDHHNNSSTTT format.

Initial Time Offset From UTC: It is a MANDATORY field, since this can be used as a fallback while GPS fails to lock/respond. Operator shall configure it and while ENB is configured to use the configured date and time this value shall be used to set the UTC based offset. (+/-)HHNN format.

EnodeB

eNodeB Identity: eNodeB Identity as defined in 36.413, 9.2.1.37 Global eNB ID (20 bits). This identifies the eNB uniquely within the PLMN. This is combined with CellTable->CellId to get the 28bit Cell Identity for a cell.

eNodeB Name: eNodeB name. This is sent by eNodeB to MME in S1 Setup procedure.

Use Static IP Address: If FALSE, ENB will get IP Address using DHCP. If TRUE, ENB will use the static IP address configured below.

eNodeB IP Address: eNodeB IP (v4) address in dotted notation, to be used at backhaul port. eNodeB first tries to aquire IP address using DHCP. If it cannot, then it tries to use this address. This should be set to '0.0.0.0' if static IP is not to be used at all.

netmask address: Netmask address to configure static IP

Registration Accepted: This parameter can only be set from SNMP. TRUE means ENB is registered to the EMS (enbRegConfig.emsAddr). It is non-persistent. At reboot/restart, ENB sets it to FALSE and re-registers.

EMS Acknowledged [Configuration Change]: TRUE means any change performed via non-EMS interfaces (CLI/Web) is acknowledged by EMS. When any configuration parameter is modified through CLI/Web this is set to FALSE and a notification is sent to EMS. EMS needs to set this value to TRUE after acknowledging the change towards configuration data synchronization. It is non-persistent. At reboot/restart, ENB sets it to TRUE.

MME

MME IP Address: MME IP (v4) address.

Secure MME Link Usage: TRUE means S1 signalling is to be protected by IPSec

Secure SGW Link Usage: TRUE means S1 user-plane is to be protected by IPSec

Number of Outbound SCTP Streams for S1: Number of SCTP Streams for S1. 1 Stream refers to a pair of 1 inbound and 1 outbound. 1 Stream will be used for Non-UE-Specific Signaling. Others will be used for UE-Specific Signaling.

Cell

Cell Identity: Cell Identity. The final value of Cell Identity is constructed by taking 20MSB from eNBConfig->eNBIdentity and 8 LSB from CellId

Cell Type: Verysmall is Pico or Femto.

MIMO Enabled: Parameter to specify whether ENB uses MIMO or not. If TRUE, ENB will enable MIMO in Scheduler. Otherwise, MIMO will not be used.

Cell Barred: This parameter defines whether the cell is barred. ‘barred’ means the cell is barred

MCC: Mobile country code (3 digits)

MNC: Mobile network code. It is 2 or 3 digits

Tracking Area Code: Tracking Area Code for the cell

Frequency Band Indicator: Frequency Band Indicator for FDD mode

Downlink BandWidth: Dl Channel bandwidth. Range is 5,10, 20 MHz

Downlink EARFCN: ARFCN for downlink E-UTRA carrier frequency. The number is as per the band configured for the cell using the table defined in 36.101. Valid Range: 5180 - 5279

Uplink Bandwidth: UL Channel bandwidth. Range is 5,10, 20 MHz .This also translates into RB configuration : Value 5MHz corresponds to 25 resource blocks, 10MHz to 50 resource blocks and so on. If for FDD this parameter is set to NotDefined, the uplink bandwidth is considered equal to the downlink bandwidth and corresponding IE is not broadcast in SIB2 in the cell

Uplink EARFCN: ARFCN for uplink E-UTRA carrier frequency. The number is as per the band configured for the cell using the table defined in 36.101. If absent, the (default) value determined from the default TX-RX frequency separation defined in TS 36.101 [42, table 5.7.3-1] applies. Valid range : 23180 - 23279

Cell reserved for Operator use: This parameter specifies whether the cell is reserved for operator use or not.

Max Connected mode Ues: Maximum number of connected mode UE in the cell. Used for admission control. Limit can be subject to licensing.

Access Barring for Emergency Calls: Access class barring for AC 10. FALSE means access allowed.

AC Barring MO Signaling: AC Barring used for MO signaling. If set to FALSE< the AC barring for MO Signaling data is not included in SIB2.

AC Barring Factor MO Signaling: Access barring factor for mobiled originated Signaling. If the random number drawn by the UE is lower than this value, access is allowed. Otherwise the access is barred. The values are interpreted in the range [0,1): p00 = 0, p05 = 0.05, p10 = 0.10,…,p95 = 0.95.

Mean AC Barring Time MO Signaling: Mean access barring time for mobile originated signaling, in seconds

Barring AC 11-15 MO signaling: Access class barring for AC 11-15 for mobile originated signaling. The first/ leftmost bit is for AC 11, the second bit is for AC 12, and so on.

AC Barring MO Data: AC Barring used for MO data calls. If set to FALSE< the AC barring for MO data callsis not included in SIB2.

AC Barring Factor MO Data: Access barring factor for mobiled originated data calls. If the random number drawn by the UE is lower than this value, access is allowed. Otherwise the access is barred. The values are interpreted in the range [0,1): p00 = 0, p05 = 0.05, p10 = 0.10,…,p95 = 0.95.

Mean AC Barring Time MO Data: Mean access barring time for mobile originated data calls, in seconds

Barring AC 11-15 MO Data: Access class barring for AC 11-15 for mobile originated data calls. The first/ leftmost bit is for AC 11, the second bit is for AC 12, and so on.

Lower Bound Value for RNTI Pool: Lower bound value for RNTI allocated in the cell. RNTIs allocated from this pool are RA-RNTI, C-RNTI, SPS C-RNTI & Temporary C-RNTI

RNTI Pool Size: Size of RNTI pool for the cell. It should be ensured that the size is sonfigured so that startRnti + sizeRntiPool < 65536

RSRP Measuemenr Filter Coefficient: Specifies the filtering coefficient for RSRP measurements used to calculate path loss, which is used for uplink power control.

MAC Profile Reference: Scheduler Profile Identifier reference. This is the Profile Id of the SchedulerConfig table that is mapped to this cell (for MAC/scheduler configuration)

PHY Profile Reference: PHY Profile Identifier reference. This is the Profile Id of the PHYConfig table that is mapped to this cell (for PHY configuration)

Physical Cell Identity: Physical Cell-Identity used at physical layer. This together with PhysicalCellGroupId makes the unique physical cell Identity (0..504). Each cell within one eNB must have unique physical cell ID and must be separated from neighboring eNB cells with same frequency.

Physical Cell Group Identity: Physical Cell-group Identity used at physical layer. This together with PhysicalCellId makes the unique physical cell Identity (0..504). Each cell within one eNB must have unique physical cell ID and must be separated from neighboring eNB cells with same frequency.

P-Max: The parameter P-Max is used to limit the UE's uplink transmission power on a carrier frequency and is used to calculate the parameter Pcompensation defined in TS 36.304. Corresponds to parameter PEMAX_H in TS 36.101.

UE Response Timer: Timer to supervise the RRC message from UE in response to the RRC message sent by eNodeB. The values are in terms of ms, i.e. ms50 means 50ms, ms100 means 100ms and so on.

RRC Connection Reject Wait Time: Time for which UE should wait before re-trying RRC Connection attempt. Value is in seconds.

UE Inactivity Check Interval: The interval in secords at which ENB shall check for UE-inactivity. If there is (no UL data from UE to ENB) and (no DL data from SGW to ENB for that UE) for this duration, the UE will be considered to be 'inactive'. This timer should be interpreted as 'inactivity for atleast this duration', rather than 'inactivity for exactly this duration'.

SIB1 Profile Reference: SIB1 Profile Reference. This defines the SIB1 Profile (configuration table) that is used by this cell

SIB2 Profile Reference: SIB2 Profile Reference. This defines the SIB2 Profile (configuration table) that is used by this cell

SIB3 Profile Reference: SIB3 Profile Reference. This defines the SIB3 Profile (configuration table) that is used by this cell

Trace Control

Trace Command: Allows operator to start and stop message tracing in eNodeB. The 'duration' field indicates the period for which traces should be collected. eNodeB starts collecting the trace after receiving this command and continues to collect for 'duration' minutes. At end of 'duration', eNodeB stops the trace and eNode generates an alarm. The alarm contains the trace-file name. The objects to be traced are configured in TraceConfig Table. Traces are collected in a single-file in eNodeB. Trace is a command; it does not persist across reboot, restart, activation. Operator can stop an ongoing trace by sending stopTrace command. The 'duration' field in stopTrace command is 'dont care' i.e. ignored by eNodeB.

Trace Duration: Gives the duration in minutes for which the traces should be collected. This field is ignored for stopTrace command.
