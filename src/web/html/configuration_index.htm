Configuration Management

Configuration Management is responsible for editing and retrieval of configuration information for the various resources of eNodeB system. The resources may be physical (e.g. Physical Layer) or logical (e.g. Cell). The actual resource types that are managed are eNodeB specific. Present configuration data for any eNodeB resource is retrieved and displayed before any modification, so that the operator can selectively change the data items that need to be updated. For performing configuration, the operator first decides the resource on which the operation is to be performed.

Most configuration commands operate on only one resource at a time. For some resources or their parameters its a requisite that eNodeB should be in LOCKED state, such parameters are disabled for editing in case eNodeB is in UNLOCKED state.



Configuration of following is supported.

- eNodeB Parameters
- MME Parameters
- Cell Parameters
- Physical Layer Parameters
- SIB1 & SIB2 & SIB3 Parameters
- Scheduler Parameters
- Quality of Service Parameters
- Message Tracing Parameters
- Logging Parameters
- Radio Head Parameters
- Trace Collector Parameters

- Timers Parameters
- EUTRA Profile/Neighbour Parameters
- Event Threshold Parameters
- eNodeB Address Parameters
- SRS Dedicated Parameters
- Security Configuration
- Counters Configuration
