Monitor (Status/Performance)

Monitoring is primarily responsible for retrieving the present status of various resources of eNodeB or of the entire eNodeB system. Performance counters are also part of the monitoring objects which can be fetched from eNodeB at periodic intervals as set by the operator. Various eNodeB resources (e.g. processes, alarms, etc.) or commands (e.g. Download, Upload, etc.) can be queried for their present status.

After the operator sets up the periodic refresh rate, the data is pulled from eNodeB for the status or counter data till the monitoring resource web page is open. Alarm severity level can be specified as filtering criteria for alarm monitoring. Facility is provided for the operator to save the data on the PC as flat files for future analysis.



Monitoring of following is supported.

- Status of eNodeB
- Status of eNodeB Modules
- Status of File Download/Upload
- Status of Software Activation
- Status of Currently Installed Software
- Status of eNodeB Hardware
- Status of Radio Head
- Status of Trace
- History of File Events
- Status of Active UEs
- History of UE Events
- Status of Active RAB
- History of RAB - Status of Performance Log File
- Status of GPS
- Status of eNodeB Device LEDs

- Status of Module Log File
- Status of Active Alarms
- History of Alarms
- Status of Location
- MAC General/User Plane Counters
- RLC General Counters
- PDCP General/User Plane Counters
- GTP General Counters
- SCTP General Counters
- RRC General/Cell Counters
- RRM General Counters
- S1AP General/Peer Node Counters
- ETHERNET Counters
- IP Counters
- Cell Status
